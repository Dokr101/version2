⚠️ Issues & Concerns

Critical Issues High Priority

  • Database Credentials Exposed: Database username (root) and empty password in config.php should be moved to environment variables (.env file) or kept outside the web root
  • Khalti Keys in Code: Payment gateway keys are hardcoded in khalti_config.php. These should be stored in environment variables
  • Error Details Exposed: Database connection errors reveal full error messages to users. Use generic error messages in production
  • No HTTPS Enforcement: Payment processing without SSL/TLS is a major security risk. Khalti production mode requires HTTPS

Medium Priority Issues Medium

  • No CSRF Protection: Forms lack CSRF tokens, making them vulnerable to cross-site request forgery attacks
  • No Input Validation: Limited server-side validation beyond required fields. Need validation for email format, phone numbers, date ranges, etc.
  • Session Security: Missing session security settings like session.cookie_httponly, session.cookie_secure, and session regeneration on login
  • No File Upload Limits: If room images are added, need proper validation for file types and sizes
  • No Rate Limiting: Login and signup forms vulnerable to brute force attacks without rate limiting
  • Missing Data Sanitization: While using prepared statements, output should still be sanitized with htmlspecialchars() consistently

Low Priority Issues Low

  • No Email Verification: Users can register without email confirmation
  • No Password Reset: Forgot password functionality is missing
  • Hardcoded URLs: Base URL is hardcoded throughout files instead of using a constant
  • No Logging: No audit trail or logging for important operations (bookings, payments, admin actions)
  • No Backup System: No automated database backup mechanism

🎯 Recommendations

1. Security Enhancements (Critical)

Environment Variables

Create a .env file for sensitive configuration and use a library like vlucas/phpdotenv:

DB_HOST=localhost
DB_NAME=HRMS_9
DB_USER=your_user
DB_PASS=strong_password
KHALTI_PUBLIC_KEY=your_public_key
KHALTI_SECRET_KEY=your_secret_key
CSRF Protection

Add CSRF tokens to all forms. Example implementation:

// Generate token: $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
// Verify token: hash_equals($_SESSION['csrf_token'], $_POST['csrf_token'])
Session Security

Add these settings in config.php before session_start():

ini_set('session.cookie_httponly', 1);
ini_set('session.cookie_secure', 1);
ini_set('session.use_strict_mode', 1);
// Regenerate session ID on login: session_regenerate_id(true);
HTTPS Required

Deploy with SSL certificate (Let's Encrypt is free). Add redirect in Apache/Nginx config or .htaccess

2. Input Validation & Sanitization

Server-side Validation

Create a validation helper file with functions for:

  • Email validation: filter_var($email, FILTER_VALIDATE_EMAIL)
  • Phone number format validation
  • Date range validation (check-in before check-out)
  • Price validation (positive numbers only)
  • Guest count validation (reasonable limits)
Output Sanitization

Always use htmlspecialchars($data, ENT_QUOTES, 'UTF-8') when displaying user input. You're already doing this in many places, but ensure it's consistent everywhere.
