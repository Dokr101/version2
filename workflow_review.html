🚨 Critical Issues

1. Room Status Not Synchronized

Issue: Room status in rooms table may not properly update during check-in/check-out

Impact: Rooms may show as available when occupied, leading to double bookings

Recommendation:

  • Add database trigger or update query to set room status = 'occupied' on check-in
  • Set room status = 'available' on check-out
  • Verify this logic exists in staff_checkin.php and staff_checkout.php

2. No Email Notifications

Issue: System lacks email notifications for critical events

Impact: Poor user experience, guests not informed of booking status

Recommendation: Implement email notifications for:

  • Booking confirmation (after payment)
  • Check-in reminder (1 day before)
  • Booking cancellation
  • Staff approval status

⚠️ Warnings & Important Issues

1. Payment Failure Handling

Issue: Unclear what happens if Khalti payment fails or is cancelled

Recommendation:

  • Ensure payment_error.php handles all failure scenarios
  • Allow guests to retry payment from My Bookings
  • Set timeout for pending payments (auto-cancel after 24 hours)

2. Booking Cancellation for Paid Bookings

Issue: System allows cancellation after payment without refund mechanism

Current State: Price hidden for cancelled bookings, payment status shows "N/A"

Recommendation:

  • Implement refund workflow or cancellation policy
  • Add cancellation deadline (e.g., 24-48 hours before check-in)
  • Consider partial refund based on cancellation time

3. Staff Approval Workflow Not Clear

Issue: Staff registration requires admin approval but no admin interface to manage it

Recommendation:

  • Create admin page to view pending staff accounts
  • Add approve/reject functionality
  • Notify staff member when approved or rejected

4. No Booking Modification After Creation

Issue: Guests cannot modify booking dates or guest count after creation

Recommendation:

  • Add "Modify Booking" option for pending bookings
  • Allow date changes if room is available
  • Recalculate price if changes made

5. Missing Late Check-out Handling

Issue: No mechanism for guests who don't check out on time

Recommendation:

  • Send reminder on check-out date
  • Allow staff to mark as "overdue"
  • Consider additional charges for late check-out
